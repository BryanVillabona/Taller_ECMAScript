Qué es ECMAScript

ECMAScript específicamente es el estándar que a partir del año 2015 a la actualidad se encarga de regir como debe ser interpretado y funcionar el lenguaje JavaScript, siendo este (JS – JavaScript) interpretado y procesado por multitud de plataformas, entre las que se encuentran los navegadores web, NodeJS u otros ambientes como el desarrollo de aplicaciones para los distintos sistemas operativos que actualmente existen en el mercado. Los responsables de dichos navegadores y JavaScript deben encargarse de interpretar el lenguaje tal como lo fija ECMAScript.

La lista de versiones de ECMA Script es larga. A continuación se muestra cuáles han existido a lo largo de los años.

Diferencias notorias entre versiones

Las diferencias entre las versiones de ECMAScript son significativas, ya que cada versión introduce nuevas características y mejoras al lenguaje. Algunas de las diferencias notables incluyen:

  • ES5: Introducción de "strict mode", que permite escribir un código más seguro y limpio.
  • ES6: Introducción de clases, módulos, promesas, let/const, y muchas otras características modernas.
  • ES7: Introducción de "Array.prototype.includes" y el operador de exponenciación.
  • ES8: Introducción de "async/await", que facilita la programación asíncrona.
  • ES9: Mejoras en "rest/spread properties" y "Promise.prototype.finally".
  • ES10: Introducción de "Array.prototype.flat" y "Object.fromEntries".
  • ES11: Introducción de "BigInt" para trabajar con números enteros grandes.
  • ES12: Introducción de "String.prototype.replaceAll" y "WeakRefs".
  • ES13: Introducción de "class fields" y "top-level await".
  • ES14: Introducción de "Array.prototype.findLast" y "WeakRefs".
  • ES15: Propuestas en discusión, incluyendo mejoras en "Array" y "Object".
  • ES16: Propuestas en discusión, incluyendo mejoras en "Array" y "Object".

Estas diferencias reflejan la evolución del lenguaje y su adaptación a las necesidades de los desarrolladores. Cada nueva versión busca mejorar la legibilidad, la eficiencia y la facilidad de uso de JavaScript, lo que lo convierte en un lenguaje cada vez más poderoso y versátil.

Es importante destacar que no todas las características de ECMAScript son compatibles con todos los navegadores o entornos de ejecución. Por lo tanto, es recomendable verificar la compatibilidad de las características que planeas utilizar en tu proyecto.

Como usar ECMAScript

Para usar ECMAScript, simplemente necesitas un entorno que lo soporte. La mayoría de los navegadores modernos y entornos de ejecución como Node.js son compatibles con las versiones más recientes de ECMAScript. Puedes escribir tu código JavaScript utilizando las características de ECMAScript y ejecutarlo en estos entornos.

Por ejemplo, si deseas utilizar la sintaxis de clases introducida en ECMAScript 2015, puedes escribir algo como esto:

                
class Persona {
                    constructor(nombre) {
                        this.nombre = nombre;
                    }

                    saludar() {
                        console.log(`Hola, mi nombre es ${this.nombre}`);
                    }
                }
                const juan = new Persona('Juan');
                juan.saludar(); // Salida: Hola, mi nombre es Juan

Este código define una clase llamada "Persona" con un constructor y un método. Luego, crea una instancia de la clase y llama al método "saludar".

Recuerda que ECMAScript es un estándar, por lo que las características específicas pueden variar según la implementación. Asegúrate de consultar la documentación de la plataforma que estés utilizando para obtener información sobre las características específicas que admite.

Ejemplos de cada version

A continuación se presentan ejemplos de código para cada versión de ECMAScript:

  • ES1: No hay ejemplos específicos, ya que es la primera versión del estándar.
  • ES2: No hay ejemplos específicos, ya que es una versión de corrección de errores.
  • ES3: Ejemplo de expresiones regulares: 
                                
const regex = /[a-zA-Z0-9]+/;
const str = "Hola123";
console.log(regex.test(str)); // Salida: true
  • ES4: No hay ejemplos específicos, ya que esta versión nunca se implementó.
  • ES5: Ejemplo de "strict mode": 
                                
"use strict";
function miFuncion() {
    x = 3.14; // Error: x no está definido
}
miFuncion();
  • ES6: Ejemplo de clases: 
                                
class Persona {
    constructor(nombre) {
        this.nombre = nombre;
    }
    saludar() {
        console.log(`Hola, mi nombre es ${this.nombre}`);
    }
}
const juan = new Persona('Juan');
juan.saludar(); // Salida: Hola, mi nombre es Juan
  • ES7: Ejemplo de "Array.prototype.includes": 
                                
const numeros = [1, 2, 3, 4, 5];
console.log(numeros.includes(3)); // Salida: true
  • ES8: Ejemplo de "async/await": 
                                
async function obtenerDatos() {
    const respuesta = await fetch('https://api.example.com/datos');
    const datos = await respuesta.json();
    console.log(datos);
}
obtenerDatos();
  • ES9: Ejemplo de "Promise.prototype.finally": 
                                
const promesa = new Promise((resolve, reject) => {
    setTimeout(() => {
        resolve('Éxito');
    }, 1000);
});
promesa
    .then((resultado) => {
        console.log(resultado); // Salida: Éxito
    })
    .finally(() => {
        console.log('La promesa se ha completado');
    });
  • ES10: Ejemplo de "Array.prototype.flat": 
                                
const arrayAnidado = [1, [2, 3], [4, 5]];
const arrayPlano = arrayAnidado.flat();
console.log(arrayPlano); // Salida: [1, 2, 3, 4, 5]
  • ES11: Ejemplo de "BigInt": 
                                
const numeroGrande = BigInt(123456789012345678901234567890);
console.log(numeroGrande); // Salida: 123456789012345678901234567890n
  • ES12: Ejemplo de "String.prototype.replaceAll": 
                                
const texto = "Hola, mundo. Hola, JavaScript.";
const nuevoTexto = texto.replaceAll("Hola", "Adiós");
console.log(nuevoTexto); // Salida: Adiós, mundo. Adiós, JavaScript.
  • ES13: Ejemplo de "class fields": 
                                
class Persona {
    nombre = 'Juan';
    saludar() {
        console.log(`Hola, mi nombre es ${this.nombre}`);
    }
}
const juan = new Persona();
juan.saludar(); // Salida: Hola, mi nombre es Juan
  • ES14: Ejemplo de "Array.prototype.findLast": 
                                
const numeros = [1, 2, 3, 4, 5];
const ultimoPar = numeros.findLast((num) => num % 2 === 0);
console.log(ultimoPar); // Salida: 4
  • ES15: Propuestas en discusión, sin ejemplos específicos.
  • ES16: Propuestas en discusión, sin ejemplos específicos.
